学习指南

计算机硬件组成

组成

  • 输入设备
  • 存储器
  • 控制器
  • 运算器
  • 输出设备

执行流程

1.  通过键盘输入字符
2.  计算机内部有键盘译码芯片，把对应位置的字符转换成二进制的0和1，存储到内存中
3.  控制器取出PC中指令，通过控制总线告诉MAR指令地址，MAR在存储器中找到这条指令，然后通过数据总线把指令放到MDR中缓存，之后通过控制总线判断是读还是写。接收到读指令之后通过数据总线把数据放到IR里面。PC值加1。这里假设一条指令长度为1字节
4.  之后解析指令操作码，进入ALU中执行，通过解析地址码把执行完的结果写出到某一个位置。
5.  控制器接收到输出设备的请求信号，运算完之后把结果发送给输出设备
6.  输出设备对结果进行处理

计算机编程语言

什么是程序

  • 程序就是计算机指令的集合

计算机语言存在的意义

  • 当这些指令越来越多，越来越复杂，编程人员记不住这些0和1，这个时候就需要计算机语言，帮助编程人员生成这堆复杂的计算机指令

编程语言发展史

  • 离计算机硬件越近的人，自己做的事情越多，对程序就更清楚，所以就鄙视不理解程序执行的人。越高级的语言编译器做的越多

  • 汇编语言

  • C，C++

    • C++是在C基础上的拓展。比如一些重复的代码，通过编译器来编写

    • C，C++通过编译器把指令转换成机器指令

    • C，C++编程有难度，一个主要的原因是内存的申请与释放

  • JAVA

    • 编译器，解释器
    • 把底层的麻烦的事情，交给解释器来做。
    • 优点：编程简单，缺点中间执行慢于C，C++程序

  • C#，VB，.NET

  • 解释型语言：编译完的字节码文件不能直接在目标机器上执行，还需要所谓的解释器来解释执行

内存的使用

内存

  • 内存的单位是字节，每个字节8比特
  • 每个运行中的程序都有4G的内存
  • 这个内存不是内存条，是空头支票

如何使用内存

#include <stdio.h>

	printf("%x \n",x);

  • 变量名，就是内存编号，只不过我们在写代码的时候不需要自己写编号，操作系统会替我们把这个具体的变量名和编号对应起来
  • 值，就是具体写到内存中的数据

大端小端存储模式

  • windows是小端存储,Mac也是
  • 大端模式：数据高位在内存低位，数据低位在内存高位
  • 小端模式：数据高位在内存高位，数据低位在内存低位
#include <stdio.h>
int main() 
{ 
  char x = 0x12; 
  short y = 0x1234; 
  int z = 0x12345678; // 12 34 56 78 
                      //0019FF24 0019FF25 0019FF26 0019FF27
                      //78 56 34 12
  printf("%p %p %p \n", &x, &y, &z); 
  return 0;
}
  • 大小端存储适用于多于一个字节的数据存储，比如说int类型
float f = 12.3f;
  • float double类型在内存中存储的时候有自己的一套编码规范，这个规范我们称为IEEE规范。并不能简单地按照大小端存储来算

原码反码补码

#include <stdio.h> int main() 
{ 
	/** * 默认数据宽度一个字节 * 0x12 原反补码 0001 0010 */

	/** * 默认数据宽度一个字节 * -0x12 原码 1001 0010 * 反码 1110 1101 * 补码 1110 1110 0xEE */

	return 0; 
}

  • 原码：最高位是符号位，其余各位为数值本身的绝对值

  • 反码：

    • 正数：反码与原码相同
    • 负数：符号位为1，其余位对原码取反

  • 补码：

    • 正数： 补码与原码相同
    • 负数： 符号位为1，其余位对原码取反

位运算符

什么是位运算

  • 程序中的所有数在计算机内存中都是以二进制形式存储的
  • 位运算就是直接对整数在内存中的二进制位进行操作

位运算到底能做什么

  • 如果只是做应用层的开发，对位运算不了解很正常
  • 计算机底层相关的领域：加密解密，安全，反病毒，反外挂等等，一定会经常用到位运算相关的知识

位运算有哪些

  • 按位与：&
  • 按位或：|
  • 按位异或：^
  • 按位取反： ~
  • 移位运算：>> , <
#include <stdio.h>

	0000 0011 ==> 0000 1100 C

	//右移:无符号补0，有符号补符号位

	1111 1111 ==> 1111 1111 FF 
	1111 1111 ==> 1111 1111 FF 
	1111 1111 ==> 0011 1111 3F 
}

动态申请与释放内存

  • 在之前的学习中，申请过数组，有放在函数外面的，也有放在函数里面的。无论放到什么位置，申请数组的时候，必须明确地告诉编译器数组长度是多少，因为编译器需要通过给定的长度确定向操作系统申请多大的一块内存。

  • 如果在程序变异的时候，并不知道要存储的数据是多少，这个时候就要用到动态申请内存。

  • 内存管理：

    • 代码区：我们写的一个一个函数放到代码区
    • 全局变量区：定义在函数外面的全局变量
    • 常量区：字符串，双引号的情况，首先在常量区申请内存。
    • 栈：函数执行的时候，也需要分配内存存放参数，局部变量

  • 堆：专门用来动态申请用的，也就是什么时候用，什么时候申请

  • 在堆中申请内存：*malloc()** 在堆中释放内存：free()

#include <stdio.h> 
#include <stdlib.h> 
//<malloc.h> <windows.h>这几个头文件都行 

struct Stu { int x; int y; }; 

void main() 
{ 
	//意味着函数执行完的之后，在堆里申请10个字节的内存 
	void* p = malloc(100); 

	//malloc返回值类型是指针类型， 
	//void*可以理解成编译器只把它当成指针看待， 
	//至于是什么类型的指针，在你真正使用的时候，再自己去转型。 
	//可以把它转成任何类型。 

	//100个int 
	int* p1 = (int*)malloc(100*sizeof(int)); 
	//释放内存 
	free(p1); 

	//200个char 
	char* p2 = (char*)malloc(200*sizeof(char)); 
	free(p2); 

	//100个结构体 
	Stu* stu = (Stu*)malloc(100*sizeof(stu)); 
	free(stu); 
}

  • 以32位的exe程序为例，每个进程都有4G内存空间，其中2G给操作系统用，剩下的2G，分给代码区，全局变量，栈，堆。所以动态申请的内存不会超过2G。

  • 申请内存需要指定大小，释放大小不需要指定，是怎么做到的？

    • 当我们使用malloc函数申请的时候，操作系统会专门记录下来从哪个位置申请内存，申请了多大一块内存。
    • 正因为有记录，所以释放的时候，自然知道释放多大的内存。所谓的释放，就是把这个记录删掉，以后这块内存就可以继续分配给别人。

  • 申请低于2G的内存，是否一定会申请成功？

  • 不一定，因为如果程序运行的多，一边申请，一边释放，不停重复，这就可能导致堆内存出现不连续的空隙。这个时候因为碎片的原因，如果想申请一块连续的大内存，就申请不下来。

  • 注意事项

    • 一定要判断malloc的返回值，返回成功才能使用
    • malloc申请的内存，必须使用free释放掉
    #include <stdio.h>
#include <windows.h>

void Fun()
{
  //函数释放的时候，栈里的p的内存自动删除了，
  //后面malloc的内存没有释放
  int* p = (int*)malloc(100*sizeof(int));
}

void main()
{
  Fun();
}
    • free之后，将指针置空，避免出现野指针
    #include <stdio.h>
#include <windows.h>

void main()
{
  int* p = (int*)malloc(100*sizeof(int));
  free(p);

  //*p = 12; //野指针

  //避免上面这种情况，将指针置空
  p = NULL;
}

文件

什么是文件，为什么使用文件？

  • 用于永久存储数据，计算机断电后仍然继续存在

文件的特点

  • 文件名
  • 文件路径
  • 文件大小(文件长度)
  • 文件属性：只读/可读可写、隐藏

文件类型

  • 文本文件: 文件当中只存储字符的ASCII码

  • 二进制文件: 使用二进制编辑工具打开之后，不仅仅是字符对应的ASCII码，还有其他的字符

  • 本质：无论哪种文件，都是以0，1的形式存在的

文件读写

C语言中用于操作文件的相关函数

//头文件
#include <stdio.h>

//相关函数
fopen, fclose, fread, fwrite, fseek, ftell ....

//文件操作流程

2. 处理文件(读，写)
3. 关闭文件
#include <stdio.h> 
#include <windows.h> 

void testWriteFile() 
{ 
	//1.打开文件 
	char szPath = "C:/aaa/Teat.txt";
	char szBuffer[0x10];
	//缓冲区，要写的数据在哪 
	FILE* pfile = NULL;
	//F12可以跟进去查看结构体 
	int nWrite = 0; 
	pfile = fopen(szPath, "wb");//wb写 rb读 

	if(pfile == null) 
	{ 
		printf("文件打开失败 \n"); 
		return; 
	} 

	//2.向文件中写数据 
	memset(szBuffer,0,sizeof(szBuffer));
	//初始化缓冲区 
	strcpy(szBuffer, "china");
	//往缓冲区写数据 
	//参数：缓冲区，一次写多少字节，写几次，要写的文件在哪里 
	nWrite = fwrite(szBuffer,1,strlen(szBuffer),p); 

	if(nWrite == 0) 
	{ 
		printf("文件写入失败"); 
		return; 
	} 

	//3.关闭文件 
	fclose(p); 
}

  • 我们这里的打开可以认为是得到了这个文件的一些信息，这些信息都存到了结构体里面了。这个结构体里面内容完全没有意义，因为它是给后面的函数用的。

  • 打开模式：

    • wb：write binary文件如果不存在，创建一个新的；如果文件存在，先清空

    • ab：append binary 在原来文件的内容后面追加

    • rb：read binary 读文件
#include <stdio.h> 
#include <windows.h> 

void testWriteFile() 
{ 
	//1.打开文件 
	char szPath = "C:/aaa/Teat.txt"; 
	char szBuffer[0x10]; 
	FILE* pfile = NULL; 

	int nRead = 0; 
	pfile = fopen(szPath, "rb");
	//read binary 
	if(pfile == null) { printf("文件打开失败 \n"); return; } 

	//2.从文件中读数据，假设文件内容123456789 
	memset(szBuffer,0,sizeof(szBuffer));//初始化缓冲区 
	nRead = fread(szBuffer,1,3,p);//123 
	if(nRead == 0) { printf("文件读取失败"); return; } 

	//再次读取 
	memset(szBuffer,0,sizeof(szBuffer)); 
	nRead = fread(szBuffer,1,3,p);//没有fseek，456 
	if(nRead == 0) { printf("文件读取失败"); return; } 

	//再次读取 
	memset(szBuffer,0,sizeof(szBuffer)); 
	fseek(p,3,SEEK_CUR); 
	nRead = fread(szBuffer,1,3,p);//有fseek，789 
	if(nRead == 0) { printf("文件读取失败"); return; } 

	//3.关闭文件 
	fclose(p); 
}

## 随机读取

*   如果想从指定的位置进行读取该怎么办？
*   比如说，电影本身就是一个文件，无论什么格式。使用播放器播放就是读取这个文件。**播放器支持任意位置播放，本质上就是从任意一个指定的位置读取数据，我们管这种叫做随机读取**

```c
fseek函数
int fseek(FILE* stream, long offset, int origin);
stream 文件指针
offset 偏移量
origin 相对哪里
返回值：0成功 -1失败
    

//跳到距离头部2字节的位置
fseek(fp, 2, SEEK_SET);
//跳到距离尾部2字节的位置
fseek(fp, 2, SEEK_END);
//跳到距离当前位置前2字节的位置
fseek(fp, -2, SEEK_CUR);
//跳到距离当前位置后2字节的位置
fseek(fp, 2, SEEK_CUR);
  • ftell函数

int nRead = ftell(p);//文件有多大

  • 正确读取文件的方式

    • fseek 不推荐反复使用，因为反复使用性能不好，而且容易出错
    • 建议申请一块和文件同样大小的内存，将文件一次读取到内存中进行处理

  • 针对大文件(2G)，可以使用内存映射的方式来实现

访问模式

  • 如果后面不加b，说明以文本文件打开，相当于wt, at, rt
  • 以二进制的形式，不会修改文件里的内容

预处理

#include <stdio.h>
#define NAME "benbenit"

int main(int argc, char * argv[])
{
  printf("%s \n", NAME); 
  return 0;
}
  • 计算机本身并不认识上面这些代码，只能认识0和1，当我们在vc6里写完这些代码，按F7的时候，就会生成一个exe文件，这个文件只包含0和1。把这些符号变成0和1的过程，我们称为编译。编译器就是专门做这个事的。
  • 在编译器真正工作之前，还有一个插曲。就是所谓的预处理器。预处理器是对源代码进行处理。比如说上面的#define，定义完它，就说明要使用预处理器，预处理器就会把后面的NAME替换成benbenit，也就是说，当真正编译的时候，会编译替换后的代码。我们管这个过程，叫做预处理。
  • 预处理器功能主要包含宏定义，文件包含，条件编译3个部分。

宏定义

  • 简单宏语法：#define 标识符 字符序列
  • 宏定义语法后面，不能加分号。而且替换是没有检查的，只有编译的时候才会发现
#include <stdio.h> 
#define NAME "benbenit" 
#define TRUE 1 
#define FALSE 0 i

nt fun() 
{ 
	return TRUE; //编译器编译之前，预处理器把TRUE换成1 
} 

int main(int argc, char * argv[]) 
{ 
	printf("%s \n", NAME); 
	return 0; 
}
  • 宏定义可以替换任何东西
#include <stdio.h> 
#define NAME "benbenit" 
#define TRUE 1 
#define FALSE 0 
#define A int fun(){ #define B return FALSE;}

A 
B 

int main(int argc, char * argv[]) 
{ 
	printf("%s \n", NAME); 
	fun(); 
	return 0; 
}
  • 带参数宏：#define 标识符(参数表) 字符序列
#include <stdio.h> 
#define NAME "benbenit" 
#define MAX(A, B) ((A)>(B) ? (A) : (B))//AB加括号是为了可读性 

int fun(int x, int y) 
{ 
	return MAX(x, y); 
} 

int main(int argc, char * argv[]) 
{ 
	printf("%s \n", NAME); 
	fun(); 
	return 0; 
}

  • 注意事项

    • 只做字符序列的替换工作，不做任何语法检查，在编译前处理
    • 宏名标识符与左圆括号之间不允许有空白符，应紧接在一起
    • 为了避免出错，宏定义中给形参加上括号
    • 多行声明时，回车换行前要加上字符\，字符\后面紧跟回车，中间不能有空格或其他字符

  • 末尾不要加分号

条件编译

#include <stdio.h>

int main(int argc, char * argv[])
{
  #if 0
      printf("========= \n");
  #endif

  return 0;
}
  • 查看反汇编，会看到上面的#if 和 #endif 没有被编译，这是因为预处理器的条件编译把上面的代码删掉了，后面编译器自然没有编译
  • 条件编译的作用就是可以选择，哪些代码需要编译，哪些不需要编译
#include <stdio.h>

int main(int argc, char * argv[])
{
  #if 1
      printf("========= \n");
  #endif

  return 0;
}
  • 修改成如上形式，预处理器就不会把源文件代码删掉，后面就会被编译器编译
  • 条件编译很有用，比如说写代码调试的时候，经常写一些print，通过print打印一些语句，判断编写的是否正确。当交给客户的时候，还要把这些调试的print删掉。而如果使用条件编译，就会很方便，写成如下形式：
#include <stdio.h>
#define DEBUG 1

int main(int argc, char * argv[])
{
  #if DEBUG
      printf("========= \n");
  #endif

  #if DEBUG
      printf("========= \n");
  #endif

  #if DEBUG
      printf("========= \n");
  #endif
  
  return 0;
}
  • 当我们测试阶段，就宏定义成1
  • 在交付的时候，把宏定义改成0就行了，#define DEBUG 0

预处理指令

#define 定义宏 
#undef 取消已定义宏 
#if 如果给定条件为真，则编译下面代码
#elif 如果前面的#f给定的条件不为真，当前条件为真，则编译下面代码 
#else 同else 
#endif 结束一个#if...#else条件编译块 
#ifdef 如果宏已经定义，则编译下面代码 
#ifndef 如果宏没有定义，则编译下面代码 
#include 包含文件
//常见的形式

int main(int argc, char * argv[])
{
  #if defined a //如果a定义了
  #undef a            //解除a定义
  #define a 200 //重新定义a
  #endif                //结束if
}
  • 宏定义就是替换，条件编译就是删除

文件包含

  • 如果项目比较大，把所有代码写到一个文件里，就比较混乱。而把所有代码写到不同文件里就比较好

#include <stdio.h>
