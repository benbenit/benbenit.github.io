c与c+的区别

  • C++是对C的拓展，C原有的语法C++都支持，并在此基础上拓展了一些新的语法：封装，继承，多态，模板，类，命名空间，成员函数等等
  • 拓展这些语法的原因，就是为了让人们使用的时候方便，更加高效。开发人员方便了，相对的编译器就要在原来的基础上多做很多事情
  • 上面这些语法如果了解了他的本质，就并不抽象。有很多书上把这些概念讲解的很抽象，那是因为没有从本质上真正去讲解这些概念是什么

总结-什么是封装

  • 我们把函数写到结构体里面的这种写法叫做封装
  • 使用封装的好处，就是我们可以写很少的代码使用结构体里面的成员

总结-什么是类

  • C++里，带函数的这种结构体，就叫类
  • 就像C里面，定义结构体不占用内存空间一样，在C++里面定义类也不占用内存空间，因为你并没有真正地使用它

总结-什么是成员函数

  • 在汇编，C语言里面，我们知道了函数的概念。现在我们把函数定义到了结构体里面，就有了一个新的名字，叫做成员函数
  • 为什么叫成员函数呢？是不是说这个函数占用的空间也在结构体里面？从前面我们对结构体大小的观察可以知道，成员函数并不占用结构体的空间
  • 通过上面函数调用的代码，我们知道在C里面调用函数，直接写就完了，因为函数名就相当于一个内存地址编号，只要你不重新编译，内存编号就不会改，用的时候直接call就行。C++里面我们写的是s.Plus，这样写编译器才能找到函数，但并不是说函数局部的了，仅仅是为了遵守C++语法规范，告诉编译器用的是哪个Plus
  • 综上，函数写在结构体内部和原来我们在C里见到的函数并没有什么区别，仍然是在代码区，仍然是一个全局变量，之所以这么写，是因为编译器的要求。编译器为什么这么要求？是对C里面不能起两个相同函数名的改进，意味着我们相同的函数名可以写多份了，只要在不同的结构体里面就可以了

类成员的访问控制

定义与实现分开

  • 代码有更好的可读性，但不是必须的
  • 如果自己写代码，放一个文件里，完全没问题，语法上完全支持。C++难学，难学在哪里？有很多知识点，如果没有代码经验，理解起来可能就非常困难，比如今天讲的类成员访问控制
  • 现在应该知道，把定义和实现分开是一种好的编程习惯。比如在实际开发中，大多数是多个人合作一起来做一个项目，这个时候，大家都遵守定义和实现分开的编程习惯，在整合代码的时候，出错的概率就低
  • 在很多人开发一个项目的时候，一种情况非常常见。就是你定义一个类给别人用，或者别人定义一个类给你用。当你定义类的时候，有些不希望别人用，仅仅是在类里面为了方便阅读写的辅助性的函数，不想让别人使用。这个时候可以加一个关键词private
struct sclass

public 与private的使用

  • 对外提供的函数或者变量，发布成public的但不能随意改动
  • 可能会变动的函数或者变量，定义成private的 这样编译器会在使用的时候做检测
  • 只有结构体内部才可以访问private的成员
  • public/private可以修饰函数也可以修饰变量
